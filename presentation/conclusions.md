> Баланс классов: 0.7% завершил / 99.3% не завершил.

## Алгоритм построения выборки для обучения модели

### 1. Маска 50% уроков
Для предотвращения утечки данных из будущего (модель должна прогнозировать на ранней стадии курса):
- Для каждого `user_id` & `course_id` уроки ранжируются по `lesson_id` ввиду отсутствия всех необходимых данных для филтрации по датам и наличия ошибок в данных (коррелирует с порядком прохождения).
- Берутся только первые 50% уроков.
- Фичи рассчитываются по данным, попавшим в маску.
- `target` рассчитывается на полных данных (по итогу курса).

### 2. Завершённые курсы
Для обучения модели используются только записи, где:
- `access_finished_at` < текущая дата (курс завершился)
- `wk_points` > 0 (была активность)

> **Итого**: порядка 61K записей из 290K для обучения.

### 3. Расчёт признаков
Агрегаты рассчитываются на трёх уровнях:
- **`user_id` & `course_id`** (с маской): из `user_lessons`, `user_trainings`, `wk_media_view_sessions`
- **`user_id`** (с маской): из `user_lessons`, `user_activity_histories`, `wk_media_view_sessions`, `user_award_badges`
- **`course_id`** (без маски, характеристики курса): из `lessons`, `lesson_tasks`, `groups`
- **`user_id`** (исторические): прогресс по завершённым ранее курсам

## Проверка признаков на утечку

- Однофакторный анализ показал AUC до 0.998 у поведенческих признаков — сильный риск утечки.
- После фильтрации оставлен набор из 7 «честных» признаков, доступных до момента построения прогноза:
  - `u_badges_cnt` — количество бейджей
  - `timezone_group` — часовой/региональный кластер
  - `u_prev_courses_cnt` — число ранее завершённых курсов
  - `with_school` — привязка к школе
  - `u_prev_progress_mean`/`u_prev_progress_max` — средний/макс прогресс по прошлым курсам
  - `u_prev_points_mean` — средние баллы по прошлым курсам

## Построение модели

### Постановка задачи и качество данных
- В `04_final_v2.ipynb` зафиксирован переход от задачи на уровне пары `student-course` к задаче на уровне студента (`успешен хотя бы на одном курсе / неуспешен`).
- Причина перехода: на уровне пар `student-course` для части пользователей встречаются одновременно `target = 0` и `target = 1`, а также наблюдается экстремальный дисбаланс, из-за которого модель склоняется к тривиальному ответу.
- После смены постановки каждому `user_id` соответствует одна метка (`1` или `0`), устраняется конфликт целевой переменной и заметно снижается шум обучающей выборки.

### Итог моделирования на априорных признаках
- LightGBM на тех же априорных признаках достигает `ROC-AUC = 0.9369` (CV), существенно лучше линейного baseline.
- Наибольший вклад в модели дают `u_badges_cnt`, `timezone_group`, `u_prev_courses_cnt`; **доминирует поведенческий сигнал "коллекционера достижений"**.
- Для M3 (`N=1785`) средняя предсказанная вероятность успеха `0.834`, медиана `0.840`; при пороге `0.80` прогнозируемая доля успешных составляет `55.1%`, а группа риска (`p < 0.70`) — `6.7%`.

### Результаты
> - **Логистическая регрессия** (baseline): **$ROC-AUC = 0.6029$**
> - **LightGBM** (градиентный бустинг): **$ROC-AUC = 0.9369$** (5-fold CV)
> - Ключевые признаки по важности: `u_badges_cnt`, `timezone_group`, `u_prev_courses_cnt`

### Предсказания для модулей 3 и 4
- Модель применена к выборкам `stats__module_3` (1785 студентов, модуль завершён) и `stats__module_4` (1707 студентов, модуль ещё идёт).
- Результат: файл `predictions_m3_m4.csv` с вероятностями и уровнями риска.

## Рекомендации бизнесу

1. **Система раннего оповещения**: интеграция модели в LMS для автоматического выявления студентов в зоне риска.
2. **Геймификация как индикатор**: количество бейджей — сильнейший предиктор успешности. Рекомендуется усилить систему бейджей и отслеживать динамику их получения.
3. **Региональная специфика**: часовой пояс/регион студента влияет на вероятность завершения — возможно, нужна адаптация расписания вебинаров / упор на студентов из определенных регионов.
4. **Онбординг для новичков**: студенты без истории на платформе (`u_prev_courses_cnt = 0`) — группа повышенного риска, требующая усиленного сопровождения.
